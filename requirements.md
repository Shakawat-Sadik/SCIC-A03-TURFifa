TypeScript Project Requirements
Build a complete, production-ready Full Stack application using TypeScript that demonstrates your understanding of frontend development, backend development, database management, authentication, authorization, API design, and professional UI/UX practices.

The goal is to build a polished application with clean architecture, reusable components, secure APIs, and excellent user experience.

1. Technology Stack
   Frontend Stack:
   React.js / Next.js
   TypeScript (mandatory)
   Tailwind CSS
   Recharts / Chart.js
   Backend Stack:
   Node.js and Express.js / Next.js API Routes (server)
   TypeScript (mandatory)
   MongoDB with JWT / Static Data with Firebase

2. Global UI & Design Rules
   Use a maximum of 3 primary colors (+ optional neutral color).
   Maintain consistent layout, spacing, and alignment throughout the project.
   All cards and components must have the same size, border radius, and visual style.
   Fully responsive for mobile, tablet, and desktop.
   No placeholder or dummy content allowed.

3. Home / Landing Page
   Navbar
   Full-width background.
   Minimum 3 routes (logged out).
   Minimum 5 routes (logged in).
   Sticky or fixed position.
   Fully responsive.

Hero Section
Height limited to 60–70% of the screen.
Interactive elements (slider, animation, CTA).
Clear visual flow to the next section.
Sections

Minimum 7 meaningful sections. Examples include Features, Services, Categories, Highlights, Statistics, Testimonials, Blogs, Newsletter, FAQ, Call to Action, etc.
Note: These are just examples; you are free to design your own sections based on project context and ideas.

Footer
Fully functional footer.
Working links only.
Contact information and social links included.

4. Core Listing / Card Section
   Each card must include: Image, Title, Short description, Meta info (price, date, rating, location, etc.), “View Details” button
   Card Rules:
   Same height and width
   Same border radius and layout
   Desktop view: 4 cards per row
   Skeleton loader while data is loading

5. Details Page
   Publicly accessible
   Multiple images or media (if applicable).
   Separate sections:
   Description / Overview
   Key information / Specifications
   Reviews / Ratings (if applicable).
   Related items (if applicable).

6. Listing / Explore Page
   Search bar
   Filtering must be implemented using at least 2 fields (e.g., category, price, rating, date, location).
   Sorting options
   Pagination or infinite scroll
   Fully functional filtering

7. Authentication System
   Login and Registration pages
   Proper validation and error handling
   Demo login button (auto-fill credentials)
   Social login (Google / Facebook) (optional)
   Clean and professional UI

8. Protected Page: Add Items (/items/add)
   Only accessible when logged in; redirect others to /login
   Form fields:
   Title
   Short description
   Full description
   Price/date/priority/relevant field
   Optional image URL
   Buttons: Submit (add)

9. Protected Page: Manage Items (/items/manage)
   List all products in a table/grid
   Each row/card with actions: View, Delete
   The layout should be clean, readable, and responsive

10. Additional Pages
    At least 2 additional pages, such as:
    About
    Contact
    Blog
    Help / Support
    Privacy / Terms

11. UX & Responsiveness
    No lorem ipsum or placeholder content
    Fully responsive across all devices
    Proper spacing and alignment
    All buttons and links must be clickable

12. Final Submission Requirements
    Live Website URL
    GitHub Repository Link (frontend and backend)
    Demo Credentials
    User Email & Password
    Admin Email & Password
